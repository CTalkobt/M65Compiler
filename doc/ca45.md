# ca45 — Mega65 45GS02 Assembler

`ca45` is a powerful, high-level assembler designed for the MEGA65 (45GS02). It combines traditional assembly with advanced procedural features, dynamic variable management, and multi-pass optimizations.

## Syntax Overview
- **Comments**: 
    - Semicolon (`;`) — Traditional 6502 style.
    - Double-slash (`//`) — C/KickAssembler style (single-line).
    - Slash-star (`/* ... */`) — C/KickAssembler style (multi-line).
- **Labels**: Defined by an identifier followed by a colon (e.g., `name:`).
    - **Scoped Labels**: `ca45` supports scoped labels where colons are used as separators. The `cc45` compiler utilizes this format (e.g., `main:L0:`) to uniquely identify temporary labels within procedures, ensuring they do not collide with labels in other scopes.
    - **Scoping Format**: The `double:colon:` style (the identifier itself containing colons) is utilized for scoping by the C compiler during the assembly phase to flatten the hierarchical scope into globally unique identifiers in the final binary.
    - **Local Labels**: Labels prefixed with **`@`** (e.g., `@loop:`) are local labels that are automatically scoped under the nearest preceding non-`@` label. This allows the same `@` label name to be reused in different routines without conflict:
        ```assembly
        _func1:
            @loop:          ; stored as _func1:@loop
                dex
                bne @loop   ; branches to _func1:@loop
                rts
        _func2:
            @loop:          ; stored as _func2:@loop (no conflict)
                dey
                bne @loop   ; branches to _func2:@loop
                rts
        ```
      Duplicate `@` labels within the same scope (i.e., under the same non-`@` label) are reported as errors. Inside `proc` blocks, `@` labels are scoped by the `proc` name instead.
      Additionally, `@` labels are optimization-transparent — they do not reset the assembler's register tracking, allowing redundant load elimination to work across `@` label boundaries.

    - **Procedure Scoping Validation**: `ca45` rigorously validates procedure blocks. It detects and reports errors for:
        - **Nested Procedures**: `proc` directives inside another `proc` block.
        - **Orphan `endproc`**: `endproc` without a preceding `proc`.
        - **Unclosed Procedures**: `proc` blocks that are never terminated at the end of the file.
- **Equates**: `NAME = value`. Supports arithmetic (`+`, `-`, `*`, `/`) and various literal formats.
- **Program Counter Reference**: `*` can be used in expressions to refer to the current program counter address (e.g., `bra *+4` branches forward 4 bytes from the current instruction, `lda *-2` references 2 bytes before the current address).
- **Local Scoping**: `ca45` supports nested local scopes for labels and variables.
    - **Procedures**: Each `proc` / `endproc` block creates a new named scope.
    - **Anonymous Blocks**: Curly braces `{ ... }` create anonymous local scopes for isolating labels within a procedure or global area.
    - **Structured Data Referencing**: By combining labels and anonymous blocks, you can create struct-like data structures that are accessible via colon-separated paths (e.g., `outer:inner:element`).
        ```assembly
        mystruct:
        {
          a: .byte $00
          b: .word $0000
        }

        lda mystruct:a    ; Accesses the byte at the start of mystruct
        ldax mystruct:b   ; Accesses the word at offset 1
        ```
    - **Resolution**: Symbols are resolved using a hierarchical lookup (Inner Scope -> Parent Scope -> Global Scope). This allows reuse of common label names (like `loop:`) without conflict.

- **Case Sensitivity**: Mnemonics and register names are case-insensitive.

## Literal Formats
- **Hexadecimal**: `$1234`
- **Decimal**: `123`
- **Binary**: `%1010`
- **Character**: `'A'` (PETSCII-encoded: lowercase `'a'`–`'z'` → `$41`–`$5A`, uppercase `'A'`–`'Z'` → `$61`–`$7A`)
- **String**: `"Hello"` (PETSCII or ASCII depending on directive)

## Expression Operators

Expressions in `ca45` support arithmetic, bitwise, comparison, and logical operators, as well as special functions for compile-time symbol resolution.

### Arithmetic Operators

- **Addition**: `a + b`
- **Subtraction**: `a - b`
- **Multiplication**: `a * b`
- **Division**: `a / b` (integer division, truncates toward zero)
- **Modulo**: `a % b` (remainder)
- **Exponentiation**: `a ** b` (power; e.g., `2 ** 8` = 256)
- **Unary Negation**: `-a`

### Bitwise Operators

- **Bitwise AND**: `a & b`
- **Bitwise OR**: `a | b`
- **Bitwise XOR**: `a ^ b`
- **Bitwise NOT**: `~a` (one's complement)
- **Left Shift**: `a << b`
- **Right Shift**: `a >> b` (logical shift)

### Comparison Operators

- **Equality**: `a == b` (returns 1 if true, 0 if false)
- **Inequality**: `a != b`
- **Less Than**: `a < b`
- **Less Than or Equal**: `a <= b`
- **Greater Than**: `a > b`
- **Greater Than or Equal**: `a >= b`

### Logical Operators

- **Logical AND**: `a && b` (short-circuit; returns 1 if both non-zero, 0 otherwise)
- **Logical OR**: `a || b` (short-circuit; returns 1 if either non-zero, 0 otherwise)
- **Logical NOT**: `!a` (returns 1 if zero, 0 if non-zero)

### Ternary Operator

- **Conditional**: `condition ? true_value : false_value`

### Special Functions

- **Symbol Definition Check**: `defined(<symbol>)` — Returns 1 if `<symbol>` is defined at assembly time (post-pass-1), 0 otherwise. Useful in `.if` directives to conditionally assemble code based on whether a label or equate exists.
  - Accepts both identifier names and instruction tokens as symbol names
  - Example: `.if defined(FEATURE_FLAG) ... .endif`
  - **Note**: Results are determined in pass 2 when all symbols have been resolved

### Program Counter Reference

- **Current Address**: `$` or `*` refers to the current program counter (e.g., `bra *+4` branches forward 4 bytes)

### Operator Precedence (High to Low)

1. `$`, `*`, function calls (`defined()`)
2. Unary operators (`-`, `~`, `!`)
3. Exponentiation (`**`)
4. Multiplication, Division, Modulo (`*`, `/`, `%`)
5. Addition, Subtraction (`+`, `-`)
6. Shifts (`<<`, `>>`)
7. Bitwise AND (`&`)
8. Bitwise XOR (`^`)
9. Bitwise OR (`|`)
10. Comparison (`==`, `!=`, `<`, `<=`, `>`, `>=`)
11. Logical AND (`&&`)
12. Logical OR (`||`)
13. Ternary (`? :`)

Parentheses can be used to override precedence: `(a + b) * c`.

## Command-Line Usage
```bash
ca45 [options] <input_file.s>
```

### Options
- `-c`: **Relocatable object mode.** Produce a `.o45` relocatable object file instead of a flat binary. Enables the use of `.global` and `.extern` directives for multi-file linking with `ln45`. Default output filename changes to `out.o45`.
- -o <file>: Specify the output filename (default: `out.bin`, or `out.o45` with `-c`).
    - If the filename ends in **`.prg`**, `ca45` will automatically prepend a 2-byte little-endian load address header to the binary.
    - The header value is inferred from the first **`.org`** address encountered in the source.
- `-L <file>`: Generate an assembly listing file. Shows addresses, generated bytes, and source code side-by-side, plus a symbol table dump.
- `-v`: Verbose mode. Prints phase status messages (preprocessing, lexing).
- `-vv`: Extra verbose mode. Includes everything from `-v` plus lexical token dumps.
- `-Dname=val`: Define a symbol. Special symbols:
    - `-Dcc45.zeroPageStart=$addr`: Set the base address for stack-relative and indirect operations (default: `$02`).
- `-Roptimizer`: Report assembler optimizer actions to stderr. Each optimization is logged with source file, line number, pass name, and action description. Example:
    ```
    opt: main.c:42: [tail-call] JSR _func + RTS → JMP (saved 1 byte)
    opt: asm:10: [jmp-bra] JMP → BRA (saved 1 byte, offset -12)
    opt: main.c:15: [redundant-load] LDA $20 eliminated (A already holds value)
    opt: main.c:8: [branch-invert] bcc + BRA → bcs (saved 2 bytes)
    opt: asm:5: [jsr-bsr] JSR _sub → BSR (position-independent)
    ```
- `--experimental`: **⚠️ HIGHLY EXPERIMENTAL — LIKELY TO BREAK CODE.** Enable experimental optimizations that are known to have bugs or incomplete implementation. Currently enables:
    - **Pass 4 (Sequence Extraction)**: Extracts duplicate instruction sequences into shared routines. Has a critical size-accounting bug (issue #89) that causes null bytes in code. **DO NOT USE IN PRODUCTION.** This flag exists for testing and development only.
    - Future experimental passes may be added here. See CLAUDE.md for full list of known issues.
- `-V` / `--version`: Display version and exit.
- `-?`: Display help and exit.

### Diagnostics
The assembler provides several diagnostic warnings and errors to assist development:
- **8-bit Operand Overflow**: Warns when a value provided for an 8-bit addressing mode (like `#immediate` or base-page) exceeds 255 (e.g., `lda #$1234`).
- **Undefined Symbols**: Reports errors for symbols that are used but never defined or declared as `.extern`.
- **Duplicate Labels**: Detects and reports multiple definitions of the same label in the same scope.
- **Procedure Scoping**: Validates that all procedures are correctly nested and closed.

### Relocatable Object Mode (`-c`)

When `-c` is specified, `ca45` produces a `.o45` relocatable object file instead of a flat binary. The `.o45` format is a 32-bit extension of the standard `.o65` relocatable format, designed for the MEGA65's 45GS02 CPU.

In this mode:
- `.global <sym>` marks symbols for export (visible to the linker)
- `.extern <sym>` declares imported symbols (resolved by the linker)
- The assembler tracks which instruction operands reference relocatable or external symbols and generates relocation entries
- Segment bodies (text, data) are extracted separately with their base addresses
- The output includes a header, option records, segment bodies, relocation tables, and import/export symbol tables

See [lib45.md](lib45.md) for the complete `.o45` format specification.

**Example:**
```bash
ca45 -c math.s -o math.o45    # Produce relocatable object
ca45 -c main.s -o main.o45    # Produce relocatable object
ln45 main.o45 math.o45 -o program.prg   # Link (future)
```

## Supported Opcodes
For a complete list of supported instructions and addressing modes, see the [45GS02 Instruction Reference](opcodes.md).

`ca45` supports the full 45GS02 instruction set, including standard 6502/65CE02 instructions and MEGA65 enhancements:
- **Z Register**: `ldz`, `stz`, `inz`, `dez`, `phz`, `plz`.
- **Word Instructions**: `phw`, `inw`, `dew`, `asw`, `row`, `bsr`.
- **Extended Branches**: 16-bit relative branches (`bsr`, `bra` relfar).
- **Stack-Relative Indirect Addressing**: `lda ($nn,SP),Y` and `sta ($nn,SP),Y` (opcodes `$E2` and `$82`). These are indirect — they dereference a pointer at SP+offset, indexed by Y.
- **Simulated Direct Stack Access**: `lda offset, s` and `sta offset, s` are synthesized using `TSX` + absolute,X indexed addressing (e.g., `TSX; LDA __sp_base+offset,X`), since the 45GS02 has no native direct stack-relative load/store. The `__sp_base` symbol defaults to `$0101` and can be overridden via `-D__sp_base=$4001` for relocated stacks.
- **Flat Indirect Addressing**: `lda [zp],z`. Supports `adc`, `and`, `cmp`, `eor`, `lda`, `ora`, `sbc`, and `sta`. Automatically emits the `eom` (`0xEA`) prefix.
- **Instruction Prefix**: `eom` (`0xEA`) is supported as a standalone mnemonic.
- **Reserved Mnemonics**: The literal `nop` mnemonic is **disallowed** in `ca45` as `$EA` is now an instruction prefix on the 45GS02. Use `eom` if you specifically want the `$EA` byte, or `clv` (`$B8`) as a safe 1-byte NOP alternative.

## Simulated Opcodes (Mega65 Hardware Assisted)
`ca45` provides high-level simulated opcodes that leverage the MEGA65's hardware acceleration or provide convenient multi-instruction sequences.

### Legend
- **Registers**: `.A`, `.X`, `.Y`, `.Z` (8-bit), `.AX`, `.AY`, `.AZ`, `.XY` (16-bit), `.Q` (32-bit).
- **Flags**: `N` (Negative), `Z` (Zero), `C` (Carry), `V` (Overflow).

---

### `mul.<width> <dest>, <source>`
Performs hardware-accelerated multiplication (`dest = dest * source`).
- **Widths**: `.8`, `.16`, `.24`, `.32` bits.
- **Modes**:
    - `dest`: Register pair (`.AX`, `.AXY`, `.Q`) or `abs` memory.
    - `source`: `#imm` (immediate value), `abs` (read from memory address), register pair, or `offset, s`.
    - **Note**: Bare numeric addresses (e.g., `$04`) are treated as memory reads, not immediate values. Use `#$04` for the literal value 4.
- **Registers Affected**: `.A` (used for transfer), `dest` registers.
- **Flags Affected**: `N`, `Z` (based on last byte stored to RAM or last register loaded).

### `div.<width> <dest>, <source>`
Performs hardware-accelerated unsigned division (`dest = dest / source`).
- **Widths**: `.8`, `.16`, `.24`, `.32` bits.
- **Modes**: same as `mul`. Bare numeric addresses are memory reads; use `#` for immediates.
- **Registers Affected**: `.A`, `dest` registers.
- **Flags Affected**: `N`, `Z`.

### `mul.s16 <dest>, <source>`
Signed 16-bit multiply (`dest = dest * source`). Takes the absolute value of both operands, performs unsigned hardware multiplication, then negates the result if the operand signs differ.
- **Modes**: `dest`: `.AX`; `source`: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.
- **Note**: Uses `$D76E` as a scratch byte for sign tracking.

### `div.s16 <dest>, <source>`
Signed 16-bit divide (`dest = dest / source`). Same sign-correction strategy as `mul.s16`. Result is truncated toward zero (C99 semantics).
- **Modes**: `dest`: `.AX`; `source`: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.

### `mod.16 <dest>, <source>`
Unsigned 16-bit modulo (remainder). Performs `div.16` then loads the hardware remainder from `$D770`/`$D771`.
- **Modes**: `dest`: `.AX`; `source`: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.

### `mod.s16 <dest>, <source>`
Signed 16-bit modulo. Sign of the result follows the dividend (C99 semantics).
- **Modes**: `dest`: `.AX`; `source`: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.

### `ldax / lday / ldaz <src>`
Loads a 16-bit word into a register pair.
- **Modes**: `#imm`, `abs`, `offset, s`.
- **Registers Affected**: `dest` pair (e.g., `.A` and `.X`).
- **Flags Affected**: `N`, `Z` (based on high byte).

### `ldx / ldy / ldz <offset>, s`
Loads an 8-bit value from the stack into a register.
- **Modes**: `offset, s`.
- **Generated Code**: Uses `lda <offset>, s` followed by `tax/tay/taz`.
- **Registers Affected**: `dest` register, `.A` (clobbered).
- **Flags Affected**: `N`, `Z`.

### `stax / stay / staz <dest>`
Stores a 16-bit word from A (low) and X/Y/Z (high) into memory.
- **Modes**: `abs`, `offset, s`.
- **Registers Affected**: `.A` (if `dest` is `offset, s`).
- **Flags Affected**: `N`, `Z` (if `dest` is `offset, s`).

### `stx / sty / stz <offset>, s`
Stores an 8-bit value from a register into the stack.
- **Modes**: `offset, s`.
- **Generated Code**: Uses `txa/tya/tza`, `tsx`, `sta abs,x`.
- **Registers Affected**: `.A` (clobbered), `.X` (clobbered).
- **Flags Affected**: `N`, `Z`.

### `add.16 / sub.16 / add.s16 / sub.s16 <dest>, <src>`
16-bit addition/subtraction.
- **Modes**:
    - `dest`: `.AX`, `.AY`, `.AZ` or `abs`.
    - `src`: `#imm`, `abs`, or register pair.
- **Registers Affected**: `dest` registers, `.A` (temp).
- **Flags Affected**: `N`, `Z`, `C`, `V`.

### `and.16 / ora.16 / eor.16 .ax, <src>`
16-bit bitwise logic.
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: `.AX`, `.A` (temp).
- **Flags Affected**: `N`, `Z`.

### `lsl.16 / lsr.16 / lsl.s16 / lsr.s16 [dest]`
16-bit logical shifts.
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### `asr.16 / asr.s16 [dest]`
16-bit arithmetic right shift (preserves sign).
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `.A` (temp), `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### `neg.16 / not.16 / neg.s16 [dest]`
16-bit negation or bitwise NOT.
- **Modes**: `.ax`, `abs`, `offset, s`.
- **Registers Affected**: `dest` registers, `.A`, `.X` (if stack).
- **Flags Affected**: `N`, `Z`, `C` (if `neg`).

### `cmp.16 / cmp.s16 .ax, <src>`
16-bit comparison (unsigned or signed).
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: `.A` (temp).
- **Flags Affected**: `N`, `Z`, `C`.

### `abs.16 / abs.s16 [dest]`
16-bit absolute value.
- **Modes**: `.ax`, `abs`, `offset, s`.
- **Registers Affected**: `dest` registers, `.A`, `.X` (if stack).
- **Flags Affected**: `N`, `Z`, `C`.

### `rol.16 / ror.16 / rol.s16 / ror.s16 [dest]`
16-bit rotate through carry.
- **Modes**: `.ax`, `abs`.
- **Registers Affected**: `dest` registers.
- **Flags Affected**: `N`, `Z`, `C`.

### 32-bit Operations

The following simulated opcodes operate on the `.Q` register quad (A=byte0, X=byte1, Y=byte2, Z=byte3) or absolute memory addresses. They complement the native Q register instructions (`adcq`, `sbcq`, `ldq`, `stq`, etc.) for cases requiring register-only or special addressing modes.

### `add.32 / sub.32 / add.s32 / sub.s32 <dest>, <src>`
32-bit addition/subtraction.
- **Modes**: `dest`: `.Q` or `abs`; `src`: `#imm`, `abs`.
- **Registers Affected**: `.Q` (when dest is `.Q`) or memory (when dest is `abs`).
- **Flags Affected**: `N`, `Z`, `C`, `V`.

### `and.32 / ora.32 / eor.32 .Q, <src>`
32-bit bitwise logic.
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: `.Q`.
- **Flags Affected**: `N`, `Z`.

### `cmp.32 / cmp.s32 .Q, <src>`
32-bit comparison (unsigned or signed). Compares byte-by-byte from low to high; flags from the highest differing byte give the correct result.
- **Modes**: `#imm`, `abs`.
- **Registers Affected**: None (`.Q` preserved).
- **Flags Affected**: `N`, `Z`, `C`.

### `neg.32 / not.32 / neg.s32 [dest]`
32-bit negation (two's complement) or bitwise NOT (one's complement).
- **Modes**: `.Q`, `abs`.
- **Registers Affected**: `.Q` (when dest is `.Q`) or memory.
- **Flags Affected**: `N`, `Z`, `C` (if `neg`).

### `abs.32 / abs.s32 [dest]`
32-bit absolute value. Negates if sign bit set; no-op if positive.
- **Modes**: `.Q`, `abs`.
- **Registers Affected**: `.Q` (when dest is `.Q`) or memory.
- **Flags Affected**: `N`, `Z`, `C`.

### `lsl.32 / lsr.32 / rol.32 / ror.32 / asr.32 [dest]`
32-bit shifts and rotates. LSL/ROL shift low→high; LSR/ROR shift high→low. ASR preserves sign bit.
- **Modes**: `.Q`, `abs`.
- **Registers Affected**: `.Q` (when dest is `.Q`) or memory.
- **Flags Affected**: `N`, `Z`, `C`.
- Signed variants: `lsl.s32`, `lsr.s32`, `rol.s32`, `ror.s32`, `asr.s32`.

### `sxt.16`
Sign-extend 16-bit value in `.AX` to 32-bit `.Q`. If bit 7 of X is set, Y and Z are filled with `$FF`; otherwise `$00`. A and X are preserved.
- **Registers Affected**: `.Q` (Y and Z set; A and X unchanged).
- **Flags Affected**: `N`, `Z`, `C`.

---

### `chkzero.8 / chkzero.16 / chknonzero.8 / chknonzero.16`
Zero-detection boolean result.
- **Registers Affected**: `.A` (contains 1 or 0), `.X` (if 16-bit).
- **Flags Affected**: `N`, `Z`.

### `branch.16 <condition>, <target>`
16-bit branching (beq/bne).
- **Registers Affected**: `.X` (temp).
- **Flags Affected**: None (preserves incoming flags except via internal CMP).

### `select <reg>, <val_if_true>, <val_if_false>`
Conditional assignment based on Zero flag.
- **Registers Affected**: `reg`.
- **Flags Affected**: `N`, `Z`.

### `ptrstack <operand>`
Load address of a variable into `.AX`.
- **Stack variables** (`.var`-defined): computes `SP + __sp_base + offset` at runtime via `TSX; TXA; CLC; ADC #addr...` (9 bytes).
- **Global variables** (labels): loads the absolute address immediately via `LDA #lo; LDX #hi` (4 bytes).
- **Registers Affected**: `.AX` (and `.X` clobbered for stack path).
- **Flags Affected**: `N`, `Z`, `C`.

### `ptrderef <zp_addr>`
De-reference 16-bit pointer in Zero Page into `.AX`.
- **Registers Affected**: `.AX`, `.Y`.
- **Flags Affected**: `N`, `Z`.

### `ldw.f / stw.f <addr28>`
28-bit linear memory 16-bit load/store.
- **Registers Affected**: `.AX`.
- **Flags Affected**: `N`, `Z`.

### Frame-Relative Pseudo-ops (`.fp`)

These opcodes access pre-allocated local frame slots using `TSX` + absolute,X indexed addressing, similar to stack-relative ops but offset from the frame base (`_fp`).

Frame-relative operands support both **numeric offsets** and **symbolic variable names**:
- **Numeric**: `.fp 0x10` — Direct byte offset from frame base
- **Symbolic**: `.fp var_name` — Named frame variable (resolved via symbol table lookup). Variable name must be declared with `.local var_name = offset`

### `lda.fp <offset|name>` / `sta.fp <offset|name>`
8-bit frame-relative load/store. Generated code: `TSX; LDA/STA __sp_base+_fp+offset,X`.
- **Operand**: Either a numeric offset or a variable name declared with `.local`
- **Registers Affected**: `.A`, `.X` (clobbered by TSX).
- **Flags Affected**: `N`, `Z`.

### `ldax.fp <offset>` / `stax.fp <offset>`
16-bit frame-relative load/store into/from AX.
- **Registers Affected**: `.AX`, scratch ZP (for byte ordering).
- **Flags Affected**: `N`, `Z`.

### `leax.fp <offset>`
Load effective address of a frame slot into AX. Computes the runtime stack address of the frame variable.
- **Registers Affected**: `.AX`.
- **Flags Affected**: `N`, `Z`, `C`.

### `move.fp <dest>, <src>, <len>`
DMA block copy between two frame-relative addresses. Used for struct copies within the frame.
- **Registers Affected**: None (DMA).
- **Flags Affected**: None.

### `lda.sp / sta.sp <offset>`
8-bit stack-relative load/store. Used by the compiler for `char`-sized local variables and parameters.
- **Generated Code**: `TSX; LDA/STA __sp_base+offset,X`.
- **Registers Affected**: `.A`, `.X` (clobbered by TSX).
- **Flags Affected**: `N`, `Z`.
- **Inline ASM**: `__asm__("lda.sp _p_mychar")` or `__asm__("sta.sp _l_mybyte")`.

### `ldw.sp / stw.sp <src/dest>, <offset>`
Preferred 16-bit stack-relative load/store.
- **Registers Affected**: `.AX`, `.Y` (if `.AY`), `.Z` (if `.AZ`).
- **Flags Affected**: `N`, `Z`.

### `phw.sp <offset>`
Push a 16-bit word from the stack frame onto the stack (for passing stack-local variables as function arguments).
- **Generated Code**: `TSX; LDA __sp_base+offset+1,X; PHA; LDA __sp_base+offset,X; PHA`.
- **Registers Affected**: `.A`, `.X` (clobbered).
- **Flags Affected**: `N`, `Z`.

### `fill / fill.sp <dest>, <length>`
DMA Memory Fill.
- **Value**: taken from `.A`.
- **Modes**: `abs`, `(zp)`, register pair, `offset, s`.
- **Registers Affected**: None (preserved).
- **Flags Affected**: None.

### `move / move.sp <src>, <dest>`
DMA Memory Copy.
- **Length**: taken from `.XY`.
- **Modes**: `abs`, `.AZ`, `offset, s`.
- **Registers Affected**: None.
- **Flags Affected**: None.

### `inc.f / dec.f <addr28>`
28-bit linear memory byte inc/dec.
- **Flags Affected**: `N`, `Z`.

### `swap <reg1>, <reg2>`
Swap two 8-bit registers (`A`, `X`, `Y`, `Z`).
- **Registers Affected**: `reg1`, `reg2`.
- **Flags Affected**: None (if using stack).

### `push <reg> / pop <reg>`
Multi-register push/pop.
- **Registers**: any combination or width.
- **Flags Affected**: None (pop preserves P).

### `zero <reg1>[, ...]`
Clear registers to zero.
- **Registers Affected**: specified registers.
- **Flags Affected**: `Z` (1), `N` (0).

### `sxt.8`
Sign-extend `.A` to `.AX`.
- **Registers Affected**: `.X`.
- **Flags Affected**: `N`, `Z` (based on X).

### `expr <target>, <expression>`
Complex math expression. Supports array indexing (see `.array` directive below).
- **Array Indexing**: `expr .A, myArray[.x]` or `expr .AX, grid[.y][.x]`. Constant indices are resolved at assembly time; runtime indices generate optimized code.
- **Registers Affected**: `target`, plus stack usage.
- **Flags Affected**: `N`, `Z`, `C`, `V`.

### Bitfield Operations

These pseudo-ops extract and insert bit fields within 8-bit or 16-bit storage units (bytes or words in memory or registers).

### `bfext #bitoff, #width`
Extract an 8-bit bitfield from the A register.
- **Input**: `.A` contains the 8-bit storage unit.
- **Operands**:
    - `#bitoff` — starting bit position (0–7) within the byte.
    - `#width` — number of bits to extract (1–8).
- **Output**: `.A` contains the extracted bits (right-aligned, zero-extended). `.X` is cleared to zero.
- **Registers Affected**: `.A`, `.X`.
- **Flags Affected**: `N`, `Z`.
- **Example**: `bfext #2, #3` extracts bits 2–4 from A, placing the result in bits 0–2 of A.

### `bfext16 #bitoff, #width`
Extract a 16-bit bitfield from the AX register pair.
- **Input**: `.AX` (A=low byte, X=high byte) contains the 16-bit storage unit.
- **Operands**:
    - `#bitoff` — starting bit position (0–15) within the 16-bit word.
    - `#width` — number of bits to extract (1–16).
- **Output**: `.AX` contains the extracted bits (right-aligned). If width ≤ 8, X is cleared; if width > 8, both A and X contain the result.
- **Registers Affected**: `.AX`.
- **Flags Affected**: `N`, `Z`.
- **Example**: `bfext16 #4, #12` extracts bits 4–15 from AX, placing the 12-bit result in AX.

### `bfins addr, #bitoff, #width`
Insert an 8-bit value into a bitfield in memory using absolute addressing.
- **Input**: `.A` contains the new value to insert.
- **Operands**:
    - `addr` — memory address (zero-page or absolute) containing the 8-bit storage unit.
    - `#bitoff` — starting bit position (0–7) where the new value is inserted.
    - `#width` — number of bits to insert (1–8).
- **Operation**: Performs a read-modify-write (RMW) on the memory location. Clears the bits at positions [bitoff..bitoff+width-1], then ORs in the shifted new value.
- **Registers Affected**: `.A` (clobbered during operation).
- **Flags Affected**: None (RMW flags not exposed).
- **Addressing Modes**: Absolute (default), zero-page.
- **Example**: `lda #3; bfins $4000, #0, #2` inserts the 2-bit value 3 into bits 0–1 of memory location $4000.

### `bfins.sp offset, #bitoff, #width`
Insert an 8-bit value into a bitfield on the stack (stack-relative addressing).
- **Input**: `.A` contains the new value to insert.
- **Operands**:
    - `offset` — stack offset (from the current frame pointer or SP).
    - `#bitoff` — starting bit position (0–7).
    - `#width` — number of bits to insert (1–8).
- **Operation**: Same as `bfins`, but the storage unit is located on the stack.
- **Registers Affected**: `.A`, `.X` (clobbered).
- **Flags Affected**: None.
- **Example**: `lda #1; bfins.sp 2, #3, #1` inserts bit value 1 at bit position 3 of the stack word at offset 2.

### `bfins.ind zpaddr, #bitoff, #width`
Insert an 8-bit value into a bitfield via indirect addressing (16-bit pointer in zero-page).
- **Input**: `.A` contains the new value to insert. Zero-page location `zpaddr` must contain a 16-bit pointer (little-endian).
- **Operands**:
    - `zpaddr` — zero-page address of a 16-bit pointer to the storage unit.
    - `#bitoff` — starting bit position (0–7).
    - `#width` — number of bits to insert (1–8).
- **Operation**: Dereferences the ZP pointer and performs RMW on the targeted byte.
- **Registers Affected**: `.A`, `.Y`.
- **Flags Affected**: None.
- **Example**: If `$20` contains a pointer to a struct member, `lda #15; bfins.ind $20, #0, #4` inserts the 4-bit value 15 into bits 0–3 of that member.

### `bfins16 addr, #bitoff, #width`
Insert a 16-bit value into a bitfield in memory using absolute addressing.
- **Input**: `.AX` (A=low, X=high) contains the 16-bit new value to insert.
- **Operands**:
    - `addr` — memory address (2-byte, little-endian storage) containing the 16-bit storage unit.
    - `#bitoff` — starting bit position (0–15) where the new value is inserted.
    - `#width` — number of bits to insert (1–16).
- **Operation**: Performs RMW on both bytes of the 16-bit storage unit. Clears the target bits, then ORs in the shifted new value.
- **Registers Affected**: `.AX` (clobbered).
- **Flags Affected**: None.
- **Addressing Modes**: Absolute (default), zero-page.
- **Example**: `ldax #$0AB5; bfins16 $4000, #0, #12` inserts the 12-bit value into bits 0–11 of the word at $4000.

### `bfins16.sp offset, #bitoff, #width`
Insert a 16-bit value into a bitfield on the stack.
- **Input**: `.AX` contains the 16-bit new value to insert.
- **Operands**:
    - `offset` — stack offset to the 2-byte storage unit.
    - `#bitoff` — starting bit position (0–15).
    - `#width` — number of bits to insert (1–16).
- **Operation**: Same as `bfins16`, but the storage unit is on the stack.
- **Registers Affected**: `.AX` (clobbered).
- **Flags Affected**: None.
- **Example**: `ldax #$00FF; bfins16.sp 4, #8, #8` inserts an 8-bit value into bits 8–15 of the stack word at offset 4.

### `bfins16.ind zpaddr, #bitoff, #width`
Insert a 16-bit value into a bitfield via indirect addressing.
- **Input**: `.AX` contains the 16-bit new value. Zero-page location `zpaddr` contains a 16-bit pointer to the storage unit.
- **Operands**:
    - `zpaddr` — zero-page address of a 16-bit pointer.
    - `#bitoff` — starting bit position (0–15).
    - `#width` — number of bits to insert (1–16).
- **Operation**: Dereferences the ZP pointer and performs RMW on the targeted 2-byte word.
- **Registers Affected**: `.AX`, `.Y`.
- **Flags Affected**: None.
- **Example**: `ldax myValue; bfins16.ind $20, #4, #12` inserts a 12-bit value into bits 4–15 of the word pointed to by the pointer in $20.


## Procedures and Stack Management
`ca45` provides directives to manage 45GS02-style procedures with automated stack framing:

### `proc <name>[, [b#|w#]<param1>, ...]`
Defines a procedure and its named parameters.
- **Parameters**: Can be prefixed with `b#` (8-bit byte), `w#` (16-bit word, default), or `d#` (32-bit dword). Size prefixes are optional — when omitted, parameters default to 16-bit.
- **Prologue**: `proc` emits no code (0 bytes). It establishes a scope and creates parameter symbols.
- **Stack-param mode** (default): Parameters are assigned stack-relative offsets starting at 2 (past the 2-byte return address). Offsets are tracked via `.var` directives and automatically adjusted as local variables are declared. `arg1`, `arg2`, etc., are also available as positional aliases.
- **ZP-param mode** (`-fzpcall`): Parameters are not placed on the stack. Instead, the compiler assigns them to fixed zero-page addresses. The `proc` line still declares parameter names for scoping, but the compiler overrides their values using `.var` to point to the assigned ZP locations, emitting direct ZP access instructions instead of stack-relative ones.

### `call <name>[, <arg1>, ...]`
Invokes a procedure with automated parameter passing.
- **Arguments**: Pushed onto the stack before the `jsr`. Supports immediate values (`#$10`), labels, or existing stack variables.
- **Inference**: Size of arguments is inferred from the symbol table or forced with `b#`/`w#`.

### `endproc`
Completes a procedure block.
- **Cleanup**: Emits an optimized `rtn #n` or `rts` to return to the caller. When preceded by dead code (e.g., an explicit `rtn`), no code is emitted.

## Directives (Pseudo-ops)
- `.cpu <name>`: Selects the target CPU (e.g., `.cpu _45gs02`). Defaults to 45GS02 if unspecified or unsupported.
- `.org <addr>`: Sets the current Program Counter (PC).
- `* = <addr>`: Alternative syntax for `.org` (KickAssembler compatibility).
- `.byte <v1>[, ...]` : Emits 8-bit bytes.
- `.word <v1>[, ...]` : Emits 16-bit words (little-endian).
- `.dword <v1>[, ...]` : Emits 32-bit double words (little-endian).
- `.long <v1>[, ...]` : Alias for `.dword`.
- .float <v1>[, ...]` : Emits 40-bit (5-byte) Commodore floating point values.
- `.res <n>[, <fill>]`: Reserves `<n>` bytes of memory. If in a data or code segment, these bytes are initialized with `<fill>` (defaults to `$00`). If in a `.bss` segment, no data is emitted, but the program counter is advanced.
- `.align <n>[, <fill>]`: Aligns the program counter to the next multiple of `<n>`. An optional `<fill>` byte can be specified (default is `$00`).
- `.balign <n>[, <fill>]`: Alias for `.align`.
- `.segment "<name>"`: Switches the active segment. Subsequent code and data will be placed in this segment.
- `.segmentOrder <name1>[, <name2>, ...]`: Defines the order in which segments are emitted to the final binary. Default is `code, data, bss`.
    - **Note on Ordering**: Segments must be address-monotonic. If a segment's start address (defined by `.org`) is lower than the current emission pointer from previous segments, an error is thrown.
    - **Gap Filling**: Any gaps between segments with explicit `.org` addresses are automatically 0-filled in binary mode.
    - **Base Address**: The first `.org` encountered in the first emitted segment defines the base address of the binary without adding leading zeros.
- `.code`: Shortcut for `.segment "code"`.
- `.data`: Shortcut for `.segment "data"`.
- `.bss`: Shortcut for `.segment "bss"`. BSS segments are used for variable reservations and do not produce output in the final binary.
- `.repeat <count>` / `.endrepeat`: Compile-time loop unrolling. Duplicates the enclosed instructions `<count>` times with zero runtime overhead — no branches, no counter. The count must be an assembler expression that evaluates to a constant. Nesting is supported.
    ```assembly
    ; Unroll 8 byte copies (24 bytes of code, 0 bytes of loop overhead)
    .repeat 8
        lda (src),y
        sta (dst),y
        iny
    .endrepeat

    ; Nested: 2 × 3 = 6 copies of inx
    .repeat 2
        .repeat 3
            inx
        .endrepeat
    .endrepeat

    ; Expression count
    .repeat SCREEN_WIDTH / 8
        sta (ptr),y
        iny
    .endrepeat
    ```
- `.segment "<name>" { ... }`: Temporarily switches to the specified segment for the duration of the block.
- `.text "<string>"` : Emits a PETSCII-encoded string (case-swapped).
- `.ascii "<string>"`: Emits a raw ASCII-encoded string.
- `.basicUpstart <address>` : Emits a C64 BASIC stub (`10 SYS <address>`) to start a machine language program.
- `.import binary "<filename>"`: Includes the contents of an external binary file directly into the current segment.
- `.incbin "<filename>"`: Alias for `.import binary`.
- `.fillto <address>[, <fill>]`: Pads the current segment with `<fill>` bytes (default `$00`) until the program counter reaches `<address>`. Reports an error if `<address>` is already passed.
- `.global <sym1>[, <sym2>, ...]`: Marks symbols for export in `.o45` relocatable object files. Exported symbols are visible to the linker and can be referenced by other object files.
- `.extern <sym1>[, <sym2>, ...]`: Declares external (imported) symbols that are defined in another object file. The assembler creates a placeholder symbol (value 0) so forward references can be assembled; the linker will patch the final addresses. Each unique extern is assigned a zero-based index used in relocation entries.
- `.weak <sym1>[, <sym2>, ...]`: Marks symbols as weak exports. A weak symbol is exported like `.global`, but can be overridden by a strong definition (`.global`) from another object file at link time. If no strong definition exists, the weak one is used. Useful for providing default implementations (e.g., `crt0` stubs) that can be optionally replaced.
    - **MEGA65 Defaults**: If no `.org` has been specified prior to `.basicUpstart`, the assembler defaults the start address to **`$2001`**.
    - **Consistent Sizing**: The stub is exactly 12 bytes long.
- `.array <name>, <element_size>, <dim0> [, <dim1> [, ...]]`: Declares a multi-dimensional array.
    - Reserves `element_size × dim0 × dim1 × ...` bytes of zero-initialized storage.
    - Creates a label `name` at the array base address.
    - Defines compile-time constants for metadata and stride computation:
        - `name.__elsize` — element size in bytes.
        - `name.__dims` — number of dimensions.
        - `name.__dimN` — size of dimension N.
        - `name.__strideN` — byte stride for dimension N (row-major order).
    - Strides are computed as: `stride[i] = product(dims[i+1..]) × element_size`.
    - Arrays can be indexed in `expr` using bracket syntax: `expr .A, name[idx0][idx1]`.
    - **Examples:**
        ```assembly
        .array scores, 1, 10           ; 10-byte array (stride0=1)
        .array board, 1, 8, 8          ; 8×8 byte array (stride0=8, stride1=1)
        .array grid, 2, 4, 3           ; 4×3 word array (stride0=6, stride1=2)

        ; Constant indexing (resolved at assembly time)
        expr .A, board[2][5]           ; LDA board+21

        ; Runtime indexing
        expr .A, scores[.x]           ; LDA scores,X
        expr .A, board[.y][.x]        ; TYA; ASL; ASL; ASL; ... TAX; LDA board,X

        ; Manual stride access (equivalent)
        lda board + 2 * board.__stride0 + 5 * board.__stride1
        ```
    - **Code generation for runtime indices:**
        - `.X` register with stride=1: single `LDA base,X` instruction.
        - Power-of-2 strides: `ASL` shifts.
        - Arbitrary strides: MEGA65 hardware multiplier ($D770).
        - Multiple runtime dimensions: partial offsets accumulated on the hardware stack.
- `.var <name> = <expr>`: Defines or updates an assembler-time variable.
- `.cleanup <n>`: Adjusts the internal stack tracking for the current procedure.
- `.local <name> = <offset>`: Defines a frame-relative symbol. Used with the pre-allocated frame system (`lda.fp`, `sta.fp`, etc.) to give names to fixed frame offsets.

### Function Attribute Directives

These directives declare per-function metadata about ZP and register usage. They must appear inside a `proc`/`endproc` block. In `-c` mode, the metadata is stored in the `.o45` symbol table as a 16-byte function attribute record, enabling interprocedural optimization by the linker.

- `.zp_uses <addr1>[, <addr2>, ...]`: Declares which ZP slots this function reads as input parameters. Example: `.zp_uses $03, $04, $05, $06`.
- `.zp_clobbers <addr1>[, ...]`: Declares which ZP slots this function may overwrite. Callers must save these if they have live data there. Example: `.zp_clobbers $03, $04`.
- `.zp_release <addr1>[, ...]`: Declares ZP slots that are consumed (caller need not restore after call). Example: `.zp_release $07, $08`.
- `.reg_clobbers <reg1>[, ...]`: Declares CPU registers not preserved across the call. Example: `.reg_clobbers A, X, Y`.
- `.flag_clobbers <flag1>[, ...]`: Declares processor flags not preserved. Example: `.flag_clobbers C, N, Z`.

**Example:**
```assembly
proc _add_words
    .zp_uses $03, $04, $05, $06      ; two int params (4 bytes)
    .zp_clobbers $03, $04            ; result overwrites param 1 area
    .reg_clobbers A, X

    lda $03
    clc
    adc $05
    sta $03
    lda $04
    adc $06
    tax
    lda $03
    rts
endproc
```

Objects without function attributes are treated conservatively as "clobbers all, releases none".

## Register and Flag References
`ca45` allows referencing CPU registers and status flags directly within expressions:
- **8-bit Registers**: `.a`, `.x`, `.y`, `.z`, `.sp` (low byte).
- **16-bit Registers**: `.ax` (a=low, x=high), `.ay` (a=low, y=high), `.az` (a=low, z=high), `.xy` (x=low, y=high).
- **Special Registers**: `.pc` (Current Program Counter).
- **32-bit Q-Register**: `.q` or `.axyz` (a=byte0, x=byte1, y=byte2, z=byte3).
- **Processor Status Flags**: `p.c` (Carry), `p.z` (Zero), `p.i` (Interrupt), `p.d` (Decimal), `p.b` (Break), `p.v` (Overflow), `p.n` (Negative), `p.e` (Emulation).

## Optimizations
`ca45` uses a multi-pass approach to optimize the final binary:
- **Tiered Branch Strategy**: All branch instructions (`bra`, `beq`, `bne`, `bcc`, `bcs`, `bpl`, `bmi`, `bvc`, `bvs`) automatically select the most efficient addressing mode (8-bit relative or 16-bit relative) based on the target offset. The assembler uses a multi-pass approach to recalculate instruction sizes and addresses until they stabilize, ensuring the most compact representation without requiring manual padding.
- **JMP to BRA Conversion**: `jmp label` is automatically converted to `bra label` (8-bit or 16-bit) to enable relative shortening.
- **Redundant Load Elimination**: The assembler identifies and removes redundant `lda`, `ldx`, `ldy`, and `ldz` instructions. For example, a load immediately following a store to the same address is eliminated. Identical immediate loads are also removed.
- **Local Optimization Windows**: Labels prefixed with `@` are local labels scoped under the nearest preceding non-`@` label. They do not break register tracking, allowing for deeper optimization across these points. The same `@` label name can be reused across different routines without conflict.
- **Dead Code Removal**: Instructions appearing after a return (`rts`, `rtn`, `rti`) within a procedure block are automatically identified and removed from the final binary until the next label is encountered.
- **Return Optimization**: `rtn #0` is automatically converted to the single-byte `rts` (`0x60`) instruction.

## Implementation Details

### Major Structures
- **`AssemblerLexer`**: Converts assembly source into a stream of tokens. Supports semicolon, `//`, and `/* */` comments.
- **`AssemblerParser`**: The heart of the assembler. It performs two passes over the tokens:
    - **Pass 1**: Identifies all labels, determines the size of each instruction and directive, and builds the symbol table.
    - **Pass 2**: Resolves all expressions and symbol references, and emits the final machine code bytes.
- **`M65Emitter`**: A low-level utility class used during Pass 2 to emit specific instruction bytes based on the MEGA65 architecture.
- **`SymbolTable`**: A hierarchical map of symbol names to `Symbol` objects, supporting nested scoping.

### Register and Flag Tracking
- **Registers**: The assembler identifies and removes redundant `lda`, `ldx`, `ldy`, and `ldz` instructions within local optimization windows (between non-`@` labels). However, it does not track register state for the purpose of simplifying high-level `expr` or simulated opcodes.
- **Flags**: Processor flags are not tracked. However, flags can be used as values in the `EXPR` simulated opcode (e.g., `EXPR .A, P.C`).

### Adding Custom Opcodes
To add or modify opcodes in the assembler:
1.  **Opcode Map**: Add a new entry to the static `opcodes` map in `src/main/AssemblerParser.cpp` (within the `getOpcodeMapInternal()` function). You must specify the mnemonic, the `AddressingMode`, and the hexadecimal opcode.
2.  **Addressing Modes**: If a new addressing mode is needed, add it to the `AddressingMode` enum in `include/AssemblerParser.hpp` and update the `calculateInstructionSize()` and Pass 2 logic in `AssemblerParser.cpp`.

### Adding New Pseudo-ops (Directives)
To add a new directive (e.g., `.my_directive`):
1.  **Lexer**: The lexer automatically identifies tokens starting with `.` as directives.
2.  **Pass 1**: Add a handler for the directive in `AssemblerParser::pass1()` (within the `match(AssemblerTokenType::DIRECTIVE)` block). This is where you determine if the directive has a size (e.g., `.byte` has a size, while `.var` does not).
3.  **Pass 2**: Add a corresponding handler in `AssemblerParser::pass2()` to emit the actual data or perform the action during the final code generation.
