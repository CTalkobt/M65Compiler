# ar45 — Archiver for .o45 Object Files

`ar45` creates and manages `.lib` static library archives containing `.o45` relocatable object files. Libraries enable selective linking — `ln45` only pulls in the members needed to resolve undefined symbols.

## Usage

```bash
ar45 <command> <archive.lib> [files...]
```

## Commands

| Command | Description |
|---------|-------------|
| `c` | Create a new archive from `.o45` files |
| `t` | List archive members (name and size) |
| `x` | Extract all members to the current directory |
| `s` | Print the global symbol index |
| `a` | Add `.o45` files to an existing archive (replaces existing members with the same name) |
| `d` | Delete named members from an archive |
| `-V` / `--version` | Display version and exit |
| `-?` | Display help |

## Archive Format (`.lib`)

```
magic         4 bytes     "a45\0"
version       1 byte      $01
member_count  4 bytes LE
symbol_count  4 bytes LE
[symbol index]
  name        NUL-terminated
  member_idx  4 bytes LE
[members]
  name_len    2 bytes LE
  name        name_len bytes
  data_len    4 bytes LE
  data        data_len bytes (.o45 content)
```

The global symbol index is built automatically by scanning each member's `.o45` export table. This index is used by `ln45` for selective linking.

## Examples

### Create a library

```bash
# Compile library modules
cc45 -c string.c -o string.o45
cc45 -c math.c -o math.o45
cc45 -c io.c -o io.o45

# Bundle into a library
ar45 c stdlib.lib string.o45 math.o45 io.o45
```

### Inspect a library

```bash
# List members
$ ar45 t stdlib.lib
string.o45 (245 bytes)
math.o45 (180 bytes)
io.o45 (320 bytes)

# Show symbol index
$ ar45 s stdlib.lib
_strlen [string.o45]
_strcpy [string.o45]
_add [math.o45]
_mul2 [math.o45]
_putchar [io.o45]
_puts [io.o45]
```

### Link with a library

```bash
# Only modules needed by main.o45 are pulled in
ln45 -prg main.o45 stdlib.lib -o program.prg

# Equivalently:
ln45 -prg main.o45 -l stdlib.lib -o program.prg
```

If `main.o45` imports `_add` and `_puts`, only `math.o45` and `io.o45` are pulled from the library. `string.o45` is not included.

Chain dependencies are resolved iteratively — if `io.o45` imports `_strlen`, then `string.o45` is also pulled in automatically.

### Update a library

```bash
# Recompile one module and update the archive
cc45 -c math.c -o math.o45
ar45 a stdlib.lib math.o45
```

### Remove a module

```bash
ar45 d stdlib.lib io.o45
```

### Extract all members

```bash
ar45 x stdlib.lib
# Extracts string.o45, math.o45, io.o45 to current directory
```
